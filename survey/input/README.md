# Survey Input

This folder is for local runtime staging.

Use it for normalized manifests and target lists that a runbook or dashboard command asks for during a local run.

Tracked files in this folder should be limited to documentation or sanitized examples. Generated or operator-provided input should stay local.

Related paths:

- `survey/fixtures/` for tracked synthetic fixtures
- `survey/output/` for generated run output
- `survey/artifacts/` for generated local artifacts
