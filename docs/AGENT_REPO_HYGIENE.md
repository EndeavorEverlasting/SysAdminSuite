# Agent Repo Hygiene

## Rule

Every agent must leave the repository in a deliberate state.

Before handoff, every file must be accounted for in one of three ways:

1. committed intentionally
2. ignored intentionally in `.gitignore`
3. removed because it was scratch, generated, or accidental

No untracked files. No mystery artifacts. No abandoned generated output.

## Why this exists

Cursor-style agents, coding assistants, local test runs, browser downloads, and build tools can leave behind files that look harmless but later poison review:

- generated CSV, TXT, JSON, HTML, ZIP, XLSX, PNG, or log files
- local target lists with real hostnames, MACs, serials, users, rooms, or site names
- downloaded libraries or test fixtures
- temporary scripts
- partial build output
- copied customer/source data

These are not neutral. They are either product code, documentation, test fixtures, ignored runtime output, or trash. There is no fourth category.

## Required closeout commands

Run these before handoff:

```bash
git status --short
```

Expected result after cleanup:

```text
# no output
```

Also run:

```bash
git diff --stat
git diff --check
```

Use `git diff --stat` to verify the changed-file shape is intentional.
Use `git diff --check` to catch whitespace damage before review.

## Handling generated files

If a file is generated by a run, test, probe, dashboard action, browser download, or local experiment:

- commit it only when it is a deliberate product fixture or documentation artifact
- otherwise add a narrow `.gitignore` rule if the file pattern is expected to recur
- otherwise delete it

Do not commit live operational data unless it is sanitized and explicitly approved for repo history.

## Sensitive operational artifacts

Never commit unsanitized files containing:

- hostnames
- IP addresses tied to real locations
- MAC addresses
- serial numbers
- usernames
- room names
- department names
- ticket exports
- live tracker workbooks
- customer/vendor internal data

These belong outside git or under ignored runtime/output folders.

## Dashboard QR target files

Technician target lists belong under:

```text
dashboard/targets/
```

The folder is documented, but local target files are ignored. Only documentation/placeholders may be committed there.

A technician can place `.txt` or `.csv` target lists in that folder for local dashboard use. Those files must remain local.

## Vendor files

Vendor files are allowed only when deliberate and reviewed.

Example: if the dashboard needs an offline QR renderer, commit a reviewed file such as:

```text
dashboard/js/vendor/qrcode.min.js
```

Do not leave downloaded vendor files untracked.

## Agent handoff checklist

Before handing off, report:

```text
Branch:
Commit(s):
PR:
Files changed:
Validation run:
git status --short: clean
Ignored/generated files touched: none, or list exact ignored paths
Known gaps:
```

If `git status --short` is not clean, stop. Either commit the file, ignore it, or remove it. The work is not complete until the tree is clean.

## Standard

Clean tree is not polish. Clean tree is the receipt.
